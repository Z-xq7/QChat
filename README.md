# QChat - 仿微信分布式即时通信系统

QChat是一个基于C++和Node.js的仿微信分布式即时通信系统，提供完整的聊天、好友管理、消息传递等功能。该项目采用分布式架构设计，包含多个服务器组件和一个跨平台客户端。

## 项目概述

QChat是一个功能完整的即时通信系统，实现了类似于微信的核心功能，包括用户注册登录、好友管理、实时消息传递、聊天界面等。系统采用分布式架构，支持高并发和负载均衡，具有良好的扩展性和稳定性。

## 系统架构

QChat系统采用微服务架构，包含以下核心组件：

### 1. GateServer（网关服务器）
- **功能**：系统的HTTP入口点，处理用户注册、登录、密码重置等API请求
- **技术**：C++、Boost.Asio、JSON
- **端口**：13419
- **主要功能**：
  - 用户注册/登录/密码重置
  - 验证码获取
  - 与下游服务通信

### 2. ChatServer & ChatServer2（聊天服务器）
- **功能**：处理客户端TCP连接，实现即时消息传递
- **技术**：C++、Boost.Asio、gRPC
- **端口**：8090 (ChatServer), 8091 (ChatServer2)
- **gRPC端口**：50055 (ChatServer), 50056 (ChatServer2)
- **主要功能**：
  - 客户端连接管理
  - 消息传递和转发
  - 心跳检测
  - 好友关系管理
  - 聊天记录存储

### 3. StatusServer（状态服务器）
- **功能**：用户状态管理，负载均衡
- **技术**：C++、gRPC
- **端口**：50052
- **主要功能**：
  - 负载均衡，选择最空闲的聊天服务器
  - 用户Token管理
  - 会话状态验证

### 4. VarifyServer（验证服务器）
- **功能**：邮件验证码服务
- **技术**：Node.js、gRPC、Redis
- **端口**：50051
- **主要功能**：
  - 生成和发送邮箱验证码
  - 验证码存储和验证

### 5. 客户端（QChat）
- **功能**：用户界面，聊天功能
- **技术**：Qt/C++、QML
- **主要功能**：
  - 用户登录/注册/重置密码
  - 聊天界面
  - 好友管理
  - 消息发送和接收

## 技术栈

### 后端技术
- **编程语言**：C++ (服务器端), JavaScript/Node.js (验证服务)

### 前端技术
- **GUI框架**：Qt/C++

## 部署说明

### 环境要求
- C++17 编译器
- Qt 5.15 或更高版本
- Node.js 14 或更高版本
- MySQL 5.7 或更高版本
- Redis 5.0 或更高版本
- Boost 1.70 或更高版本
- gRPC 框架

### 部署步骤

1. **数据库配置**：
   - 安装并启动MySQL和Redis
   - 创建数据库并执行初始化脚本

2. **启动服务**：
   ```bash
   # 1. 启动VarifyServer（验证服务器）
   cd VarifyServer
   npm install
   node server.js

   # 2. 启动StatusServer（状态服务器）
   cd StatusServer
   # 编译并运行 StatusServer.exe

   # 3. 启动ChatServer（聊天服务器）
   cd ChatServer
   # 编译并运行 ChatServer.exe

   # 4. 启动ChatServer2（聊天服务器2，用于负载均衡）
   cd ChatServer2
   # 编译并运行 ChatServer2.exe

   # 5. 启动GateServer（网关服务器）
   cd GateServer
   # 编译并运行 GateServer.exe
   ```

3. **启动客户端**：
   ```bash
   cd QChat
   # 编译并运行 QChat.exe
   ```

### 配置文件说明

- `config.ini`：各服务器的配置文件，包含数据库连接、端口、服务器地址等信息
- `QChat.pro`：客户端Qt项目配置文件
- `package.json`：VarifyServer的Node.js依赖配置

## 开发说明

### 代码结构
- 服务器端代码采用C++编写，使用Boost.Asio进行异步网络编程
- 客户端使用Qt框架，采用MVC架构模式
- 服务间通信通过gRPC实现
- 数据存储采用MySQL，缓存使用Redis

### 编程规范
- 使用单例模式管理全局状态
- 采用观察者模式处理事件和信号
- 使用工厂模式创建对象
- 异步处理大量I/O操作
