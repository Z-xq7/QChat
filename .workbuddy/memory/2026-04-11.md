# 2026-04-11 工作日志

## QChat Agent v2 架构升级方案

设计了基于工业级标准的 v2 架构升级方案（agent-v2-design.md），五大核心升级点：
1. **ToolRegistry**（工具注册中心）：IAgentTool 接口 + 统一注册，零侵入扩展
2. **ContextManager**（上下文管理）：Token 预算（MAX 6000）+ 超限自动摘要
3. **AgentMemory**（分层记忆）：短期对话历史 + 长期 KV 偏好持久化
4. **ExecutionTracer**（执行链追踪）：信号回调 + UI 实时步骤展示
5. **SafetyGuard**（安全护栏）：Prompt 注入正则检测 + 工具风险评估

实现优先级：P0 = ContextManager + ToolRegistry；P1 = ExecutionTracer + SafetyGuard；P2 = AgentMemory 长期记忆 + 并行工具执行

## QChat Agent P0 模块实现完成

完成 ToolRegistry + ContextManager 两大 P0 模块的编码与集成。

### 新增文件（6个）
- **`itoolagent.h`**：IAgentTool 纯虚接口（name/description/schema/execute/riskLevel + toApiToolDef()）
- **`toolregistry.h/.cpp`**：单例注册中心，信号 sig_tool_before/after_execute 供 UI 监听
- **`contextmanager.h/.cpp`**：Token 预算（默认 6000 token）+ 滑动窗口裁剪 + 本地摘要（不调 API）
- **`agenttools_impl.h/.cpp`**：5个工具各自继承 IAgentTool，通过 static 辅助函数复用模糊查找逻辑

### 修改文件
- **`aipage.h`**：_ctxMgr(ContextManager*) 替代裸 _messages，引入 _tokenLabel，移除旧 AgentTools 成员
- **`aipage.cpp`**：registerTools() 一行注册一个工具；ToolRegistry 信号驱动 UI 气泡；ContextManager 自动管理 Token；Token 标签颜色（灰/橙/红）随使用率动态变化；系统 prompt 工具描述由 ToolRegistry.buildToolsDescription() 动态生成
- **`QChat.pro`**：注册所有新文件

### 关键设计决策
- ToolRegistry 是 QObject 单例，emit 信号 → AIPage 槽函数更新 UI，解耦执行层与 UI 层
- ContextManager Token 估算：ceil(chars / 2.5)（offline，无需 API），超 85% 阈值触发压缩
- 压缩策略：丢弃旧的 tool/tool_calls 中间消息，保留 user/assistant 文本生成本地摘要
- 新增工具只需：创建继承 IAgentTool 的类 → 在 registerTools() 加一行 registerTool()

  finish_reason == "stop" 时显示最终回复
